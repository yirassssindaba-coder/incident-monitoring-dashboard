Proyek System Monitoring & Incident Management Dashboard dirancang sebagai aplikasi web responsif berbasis Flask (Python) dengan database SQLite yang merepresentasikan sistem pencatatan dan pemantauan insiden IT sebagaimana umum digunakan dalam aktivitas IT Operations dan IT Support. Aplikasi ini bertujuan menunjukkan pemahaman konsep SLA, incident dan problem management, serta pelaporan operasional melalui pendekatan yang stabil, sederhana, dan siap kerja, dengan dukungan tampilan yang dapat diakses dengan nyaman di desktop maupun perangkat mobile. Aplikasi menyediakan beberapa layar utama, yaitu Dashboard untuk ringkasan jumlah insiden dan visualisasi severity sederhana, Incident List dengan filter service, severity, dan status, Create Incident untuk pencatatan insiden baru, Incident Detail yang memuat Root Cause Analysis (RCA) dan resolution, SLA Tracker untuk menghitung uptime berdasarkan downtime, serta Export Report ke format CSV untuk kebutuhan laporan operasional.

Untuk memudahkan pengguna non-teknis maupun HR dalam mencoba aplikasi, disediakan panduan pemakaian singkat (How to Use) yang menjelaskan alur dasar penggunaan sistem, mulai dari pembuatan insiden, pembaruan status, pengisian RCA dan resolusi, hingga perhitungan SLA dan ekspor laporan. Setiap halaman dilengkapi UI state yang jelas, meliputi idle state dengan instruksi singkat, loading state ringan saat proses data, success state saat data berhasil disimpan atau dihitung, empty state ketika database belum memiliki data, serta error state dengan pesan yang informatif apabila terjadi kesalahan input atau proses. Pendekatan ini memastikan aplikasi tetap mudah dipahami dan tidak menampilkan error mentah kepada pengguna.

Dari sisi visual, aplikasi menggunakan skema warna dengan kontras yang jelas dan konsisten, memanfaatkan warna netral untuk konten utama, warna penanda untuk status dan severity insiden, serta warna peringatan untuk error dan kondisi SLA kritis. Tata letak berbasis card, spacing yang rapi, dan tipografi yang jelas membantu menjaga keterbacaan data operasional, terutama pada layar kecil. Antarmuka dirancang mobile-first dengan navigasi sederhana dan layout yang stabil, sehingga tabel, form, dan ringkasan dashboard tetap nyaman digunakan di perangkat mobile tanpa mengorbankan fungsionalitas.

Dari sisi keamanan, meskipun aplikasi ini bersifat sederhana dan menggunakan SQLite, diterapkan kontrol akses dasar dan praktik keamanan minimal yang realistis, seperti validasi input di sisi server, pembatasan nilai severity dan status ke pilihan yang telah ditentukan, serta penanganan error terpusat menggunakan try/except untuk mencegah aplikasi crash dan kebocoran informasi teknis. Proyek ini tidak mengklaim keamanan tingkat produksi, namun menunjukkan pemahaman konsep dasar keamanan aplikasi web, error handling, dan pemisahan logika aplikasi yang baik untuk lingkungan portofolio. Seluruh query database dibatasi pada operasi CRUD sederhana tanpa join kompleks untuk menjaga stabilitas dan meminimalkan risiko kesalahan.

Pengujian dilakukan melalui berbagai skenario, seperti menambahkan insiden baru, memperbarui status dan resolusi, menghitung SLA berdasarkan input downtime, mengekspor laporan CSV, serta memastikan dashboard tetap menampilkan empty state yang informatif ketika database kosong. Dengan pendekatan ini, proyek System Monitoring & Incident Management Dashboard menjadi aplikasi yang stabil, ringan, minim risiko error, dan sangat relevan sebagai bukti kompetensi IT Operations, incident management, dan reporting untuk kebutuhan rekrutmen magang maupun entry-level IT.
